# Team

This document defines the working model for Rachel Capital.

The repository is designed so that each contributor has a clear scope, clear limits, and clear accountability.

## Team Overview

| Contributor | Role | Core Responsibility |
| --- | --- | --- |
| Rachel | Chief Investment Officer (CIO) | Own investment judgment and capital decisions. |
| ChatGPT | Investment Committee Secretary | Own analysis, review, and written investment support. |
| Codex | Chief Technology Officer (CTO) | Own repository systems, engineering, and documentation quality. |
| Future AI | Specialized Contributors | Extend the system without breaking governance or structure. |

## Rachel

| Field | Description |
| --- | --- |
| Role | Chief Investment Officer (CIO) |
| Responsibilities | Final investment decisions; capital allocation; portfolio ownership |
| Decision Authority | Final authority on investment actions and portfolio direction |

## ChatGPT

| Field | Description |
| --- | --- |
| Role | Investment Committee Secretary |
| Responsibilities | Market research; investment thesis; weekly review; monthly review; portfolio review; risk review; decision review; architecture review; project planning |
| Rules | ChatGPT should not implement engineering features. ChatGPT reviews engineering work. |

## Codex

| Field | Description |
| --- | --- |
| Role | Chief Technology Officer (CTO) |
| Responsibilities | Repository architecture; Markdown organization; Python development; automation; dashboard; GitHub workflow; documentation consistency |
| Rules | Codex should not make investment decisions. Codex builds systems. |

## Future AI

| Field | Description |
| --- | --- |
| Role | Specialized contributor to be defined when added |
| Responsibilities | Operate within an explicitly defined scope and documented workflow |
| Requirement | Any future AI contributor must be documented here before taking on recurring work |

## Collaboration Rules

- Rachel owns investment outcomes.
- ChatGPT owns investment process support and written reasoning.
- Codex owns repository maintainability and engineering execution.
- Any cross-role conflict should be resolved by documented scope, not informal chat memory.
- New contributor roles should be added only when they solve a recurring problem.

## Escalation Rules

- Investment questions escalate to Rachel.
- Research framing and review quality questions escalate to ChatGPT.
- Repository structure, automation, and workflow questions escalate to Codex.
- If ownership is unclear, document the ambiguity before work proceeds.
