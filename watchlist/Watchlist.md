# Watchlist

This file tracks only candidates that may lead to a real allocation decision.

If a name has no specific trigger, no invalidation condition, and no plausible portfolio role, it should not appear here.

No active watch candidates are currently recorded.

## Admission Standard

- Add a candidate only when there is a real question the portfolio may act on.
- Record the buy trigger in observable terms, not in vague interest.
- Record the invalidation condition so the name can be removed cleanly.
- State the intended portfolio role before the name is discussed as a holding.

## Exit Standard

- Remove a candidate when the trigger no longer matters.
- Promote a candidate to `thesis/` only when it deserves a real ownership case.
- If a watch item creates no future decision, delete it instead of preserving a shell.

## Usage Rules

- Every line should help a future buy, pass, or defer decision.
- This file is a decision queue, not an idea scrapbook.
