# Decision Log

This file records decisions that changed capital allocation, risk, or operating process.

If a decision will matter later, it belongs here. If it will never matter again, it should not be logged.

## Entry Standard

- Log only decisions that changed portfolio exposure, risk posture, or operating rules.
- Write the reason in present-tense decision language, not in hindsight language.
- Define review criteria before the outcome is known.
- Link the decision to the thesis, dashboard, account update, or mistake entry it changed.

## Review Standard

- Review the decision on the stated date or when the thesis breaks.
- Record what was correct, what was wrong, and what should change next time.
- If the same mistake repeats, add or update an entry in `docs/Mistake_Book.md`.

## ID Rule

- Use `YYYYMMDD-##` for decision IDs.
- Keep one ID per decision, even if the position later changes again.

## Scope Boundary

- Trades without portfolio significance do not belong here.
- Commentary without a decision does not belong here.
