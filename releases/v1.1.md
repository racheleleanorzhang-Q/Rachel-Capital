# v1.1 Governance

## Release Summary

Rachel Capital established the governance layer needed to manage the repository as a long-term software-like project.

The goal of `v1.1` was to define roles, workflow, contribution standards, architecture ownership, and release discipline.

## Included

- `.ai/TEAM.md`
- `.ai/WORKFLOW.md`
- `.ai/ARCHITECTURE.md`
- `GOVERNANCE.md`
- `CONTRIBUTING.md`
- GitHub pull request and issue templates

## Why It Matters

- made GitHub the system of record for repository change history
- defined how future contributors should work
- ensured architecture and workflow changes require review

## Result

Rachel Capital became a governed project rather than only a well-structured repository.
